1.ES6 生成器函数

基本使用：

  • 生成器函数正常执行，函数体内容不会被正常执行，需要通过next方法进行推进，每次执行一个next()方法，代码执行到一个yield，以此类推
  • yield 后面的值，可以被next的返回值进行接收
  • 同样 next传递过去的参数，同样可以被yield的返回值进行接收

可执行生成器：

需求：需要链式执行，方法一执行完成后，将返回值传递给方法二，方法二根据方法一的返回值再次执行成功后，将返回值传递给方法三，以此类推。（认证、登录等）

案例解析：

  • 三个getDate方法均为异步方法，通过*gen生成器函数进行渲染，但是并未执行，在run方法中，将生成器函数作为参数传递进去，首先进行第一次的调用，并未执行next方法。
  • 通过递归的思想，以及 yield返回值中的done 布尔值，来进行判断，当前生成器函数是否执行完毕，一旦执行完毕，将最终结果进行返回，否则，始终进行next方法，直到done值为true（此案例中的result.value是一个Promise对象，需要通过.then方法，确保fullfilled，进行再次next）
